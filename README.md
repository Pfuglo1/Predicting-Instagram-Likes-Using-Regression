| Column | Category | Description |

## 🔬 Full Pipeline Walkthrough

### 🧹 Part 1 — Data Understanding & Cleaning

- Loaded dataset with `pandas` (latin encoding)
- Inspected shape, column types, and null values
- **Removed duplicates** (`drop_duplicates()`)
- **Caption Feature Engineering:** Converted raw caption text → `caption_len` (word count)
  ```python
  df['caption_len'] = df['Caption'].apply(lambda x: len(x.split(' ')))
  df.drop(['Caption'], axis=1, inplace=True)
  ```
- **Hashtag Feature Engineering:** Counted hashtags per post → `hashtag_len`
  ```python
  df['hashtag_len'] = df['Hashtags'].apply(lambda x: len(x.split('\xa0')))
  df.drop(['Hashtags'], axis=1, inplace=True)
  ```
- **Hashtag Frequency Analysis:** Exploded all hashtags, computed `value_counts()` to identify most-used tags
- **Data Quality Checks:**
  - Negative value scan across all columns → none found
  - Logical consistency check: `Likes > Impressions` → 0 violations

---

### 📉 Part 2 — Outlier Detection & Treatment

- Visualized all numerical features using **boxplots**
- **Decision: Outliers retained** — dataset is only ~102 rows; removing outliers would significantly degrade model training
- Outliers here represent **viral posts** — legitimate and valuable signal, not noise

---

### 🧪 Part 3 — Hypothesis Testing & Statistical Analysis

- **Full Correlation Heatmap** (13×13, annotated, Blues palette)
- **Bar plot** of feature correlations with `Likes`
- **Binary segmentation analysis:** `From Explore` split at median (337) into `low` / `high` groups → compared mean `Likes` per group
  ```python
  df['explore_binary'] = df['From Explore'].apply(lambda x: 'low' if x < 337 else 'high')
  df.groupby(['explore_binary'])['Likes'].mean()
  ```

### 📊 Part 4 — Exploratory Data Analysis (EDA)

#### Univariate Analysis
- Subplot histograms for all columns (4×4 layout)
- Individual `histplot` for each feature
- **Observation:** Data is heavily **right-skewed** across most features

#### Bivariate Analysis
Scatter plots examining direct relationships with `Likes`:

| Plot | Finding |
|---|---|
| `Impressions` vs `Likes` | Strong positive relationship |
| `Saves` vs `Likes` | Strong positive — saves signal intent |
| `Profile Visits` vs `Likes` | Moderate positive — high-interest posts convert |

#### Multivariate Analysis
- Full **pairplot** across all features

#### Distribution Analysis & Transformation
- Checked **skewness** and **kurtosis** pre/post-transformation
- Applied **Yeo-Johnson Power Transformation** to all feature columns (target `Likes` never transformed):
  ```python
  pt = PowerTransformer(method='yeo-johnson')
  df[cols] = pt.fit_transform(df[cols])
  ```
- Plotted KDE curves for all transformed features to verify normalization

---

### ⚙️ Part 5 — Feature Engineering

Three new ratio-based features created to capture **engagement efficiency per impression**:

| New Feature | Formula | Insight |
|---|---|---|
| `save_rate` | `Saves / Impressions` | Post save-worthiness per impression |
| `share_rate` | `Shares / Impressions` | Virality rate |
| `Profile_Visit_Rate` | `Profile Visits / Impressions` | Conversion rate to profile |

Final feature set — **15 features + 1 target:**

```python
['Impressions', 'From Home', 'From Hashtags', 'From Explore', 'From Other',
 'Saves', 'Comments', 'Shares', 'Profile Visits', 'Follows',
 'caption_len', 'hashtag_len', 'save_rate', 'share_rate', 'Profile_Visit_Rate']
```

### 🧠 Part 6 — Feature Selection

### 🤖 Part 7 — Modeling

#### Train / Test Split & Scaling
```python
x_train, x_test, y_train, y_test = train_test_split(x, y, test_size=0.2, random_state=42)
scaler = StandardScaler()

#### Benchmark — All 5 Regressors

| Model | MAE ↓ | R² ↑ | MAPE ↓ |
|---|---|---|---|
| Linear Regression | 42.05 | 0.7108 | 0.1662 |
| Decision Tree Regressor | 48.67 | 0.4473 | 0.1731 |
| **✅ Random Forest Regressor** | **37.19** | **0.7155** | **0.1235** |
| KNN Regressor | 52.40 | 0.5302 | 0.1860 |
| XGB Regressor | 46.14 | 0.5620 | 0.1630 |

#### 💡 Model Selection Insights

- ✅ **RandomForestRegressor wins** — lowest MAE & MAPE, highest R²
- ❌ KNN struggles most on this small, high-variance dataset
- 📊 Linear Regression competitive on R² but underperforms on absolute error
- 🔁 XGBoost underperforms RF here — Random Forest generalizes better with ~100 rows

#### Final Model — RandomForestRegressor on Top 7 Features

```

## ✅ Key Takeaways

- `Saves` and `Impressions` are the **strongest individual predictors** of Instagram likes
- **Ratio features** (`save_rate`, `share_rate`) add meaningful signal beyond raw engagement counts
- **Yeo-Johnson transformation** effectively corrected the heavy right skew present in all features
- With only ~102 rows, **Random Forest** outperforms XGBoost by generalizing better on small, noisy real-world data
- The complete pipeline from raw CSV to **pickle-serialized model with interactive inference** is production-ready
